Dodge Z Ball

Unreal Engine 4 C++ and BP | Aug 1, 2020 - Aug 8, 2020

Dodge Z Ball, Is Arena based, first-person action game focusing on survival mode style game play, where you must manipulate a single Dodgeball to destroy your enemies and survive the dodgeball onslaught.

Previous Next

Responsibilities

- Implement responsive ball throwing through instant-feedback UI and interpolated ball movement
- Programmed dash mechanic through C++, ensuring dash position was always valid by capsule tracing in direction of movement.
- Collaborated with teammates across the world using Git and C++ to avoid merge conflicts.

Dodge Z Ball is a horde style action game where you knockout AI controlled enemies using your dodge ball. This is the first project I completed using a combination of C++ and Blueprints. This game helped me realize the importance of both C++ and Blueprints and why they coexist. Specifically while using Git for source control and having a team of programmers, using a mix of C++ and Blueprints was almost a necessity. As I soon found out, Git and binary files do not merge well together. This meant that only one person could work on a Blueprint actor at a time without losing data. In addition, Blueprints quickly got messy and took up a lot of screen space compared to traditional code. Using only Blueprints was not an option.
My time on the project was primarily focused on player gameplay and mechanics. I developed responsive dodge balls that can pierce and bounce off of enemies based on its state. In addition, I developed a dash mechanic that felt responsive, smooth, and prevented the player from getting stuck in walls through the use of capsule tracing. Finally, I worked on UI and design to polish off the game. I developed a UI system that works with the player mechanics and gives immediate feedback based on the state of the players dodge ball.

Hot Bots

Game Maker Studio GML | 2018 - 2019

Inspired by games alike, Robot Killing Game puts the user and 3 other teammates in an endless round of slaughter. Put against the odds, the player must survive and scavenge the battlefield for new items and resources to fend off the incoming waves of robots. Unlock new maps, weapons, skins, and classes as you level up and fight your way to the top. There is no end to the madness. New types of enemies appear as you battle.

Previous Next

Responsibilities

- Implement grid-based state machine AI
- Develop proof-of-concept multiplayer system using TCP protocol through a server/client infrastructure.
- Implement data-based weapon system to rapidly implement new guns to the arsenal.

Hot Bots is a collection of ideas using the Game Maker Studio engine. I had created many prototypes prior but none had come close to the quality and ambition of this game. It could be that same reason that lead to its eventual downfall. Featuring a variety of weapons, fight against a horde of robot killing machines that will gradually get more difficult the longer you survive. Hot Bots was a dream project I developed during my high school years. While it was never finished, I experimented with many tools I never thought I would learn. At one point, it was possible to host and connect to other players using a server-client TCP connection. Enemies, weapons, bullets, and destructible walls would sync over the network seamlessly. In addition, interpolated movement was implemented to fill in for lost or delayed packets. Enemies also featured dodge mechanics that would actively try to dodge the players projectiles and special abilities to attack the player with.
